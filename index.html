The Service (Pop-up) Flow

How CardConnect works as an integrated service on partner websites.

1

Detection

Context: User is on a partner's checkout page.

Trigger: Script finds a matching offer in the CardConnect DB.

2

The Pop-up

UI: A clean, branded modal appears.

Text: "Wait! Save an extra ₹3,500..."

Buttons: `Use CardConnect & Save`, `No Thanks`.

3

Authentication & Payment

Action: Clicking 'Save' opens a compact login/payment UI.

Flow: User logs in, confirms final price, and pays from wallet.

4

Backend Handshake

Action: User confirmation triggers a request to the Cardholder pool.

User View: Modal shows live status: "Finding a Cardholder..."

5

Completion

Result: E-commerce checkout page updates to "Order Placed."

User View: Pop-up shows a success message and disappears.
